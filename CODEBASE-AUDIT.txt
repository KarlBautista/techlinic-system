SECTION 4: CODE IMPROVEMENTS & ISSUES
================================================================================

--- CRITICAL BUGS ---

  1. Variable Name Mismatch (RUNTIME ERROR)
     File: Backend/controllers/dataControllers.js (~Line 157-165)
     Issue: Error handler references undefined variable "patientsInformationError"
            instead of the correct "patientsRecordError".
     Impact: If getRecords() encounters a database error, the server will throw
             a ReferenceError instead of returning a proper error response.
     Fix: Change "patientsInformationError" to "patientsRecordError".

  2. Missing Return Statements After Response
     File: Backend/controllers/dataControllers.js (Multiple locations)
     Issue: Several res.json() calls don't have a "return" statement, allowing
            code execution to continue and potentially send duplicate responses.
     Impact: "Cannot set headers after they are sent" errors in production.
     Fix: Add "return" before all res.status().json() calls.

--- SECURITY ISSUES ---

  3. No Input Validation on Backend
     Affected: insertRecord, addDiagnosis, insertMedicine, insertPersonnel
     Issue: No validation of request body fields (type, length, format).
            Relies entirely on Supabase RLS for protection.
     Fix: Add validation using a library (joi, zod, or express-validator).

  4. Debug Console Logs in Production Code
     Files: dataControllers.js, diseasesController.js
     Examples:
       - console.log('gumagana ako diseases')
       - console.log("gumaganaaa akooo")
       - Extensive debug logging with development comments
     Fix: Remove or replace with a proper logger (winston, pino).

--- PERFORMANCE ISSUES ---

  5. Hardcoded Department List (DRY Violation)
     File: Backend/controllers/analyticsController.js
     Issue: The same department array is copy-pasted in 4 separate functions
            (weekly, monthly, quarterly, yearly per-department analytics).
     Fix: Extract to a shared constant or fetch from the database.

  6. Inefficient Analytics Query — getMostUsedMedicines
     File: Backend/controllers/analyticsController.js
     Issue: Fetches the ENTIRE diagnoses table into memory, then aggregates
            medications in JavaScript.
     Fix: Use database-level aggregation (Supabase RPC or SQL view).

  7. Race Condition on Stock Updates
     File: Backend/controllers/dataControllers.js
     Issue: Stock level is read, decremented in JS, and written back.
            Two concurrent requests could read the same stock level
            and cause incorrect inventory counts.
     Fix: Use atomic database operations (e.g., Supabase RPC with
            UPDATE ... SET stock_level = stock_level - $quantity).

  8. moment.js is Heavyweight
     File: Backend/controllers/analyticsController.js
     Issue: moment.js (~70KB) is in maintenance mode (no new features).
     Fix: Consider migrating to day.js (~2KB, same API) or native Date.

--- STRUCTURAL IMPROVEMENTS ---

  11. Inconsistent Error Response Format
      Issue: Backend responses use different shapes for errors:
        - Some: { success: false, error: "message" }
        - Some: { success: false, data: null, error: "message" }
        - Some: just log and don't respond
      Fix: Standardize all error responses with a consistent format.

  12. Silent Error Swallowing — Patient Duplicate Insert
      File: Backend/controllers/dataControllers.js (~Line 60-68)
      Issue: When patient insert fails (e.g., duplicate), the error is
             logged but processing continues silently. This could create
             orphaned records.
      Fix: Handle duplicates explicitly (upsert or proper error return).

  13. Timezone Hardcoding
      File: Backend/controllers/analyticsController.js
      Issue: UTC+8 timezone offset is hardcoded. Won't work if deployed
             in a different timezone or if users are in different regions.
      Fix: Use server timezone config or user-specific timezone from profile.

  14. Missing .env File in Frontend

      - Consider adding frontend/.env to .gitignore if not already.
      - Consider adding a .env.example file for documentation.

  15. CORS Configuration
      File: Backend/index.js
      - Verify that CORS origin is properly restricted in production.
      - Currently allows requests during development but should be
        tightened for production deployment.

  16. Date Formatting Duplication (Frontend)
      Issue: formatDate() is defined locally in multiple files:
        - PatientCountsChart.jsx
        - Dashboard.jsx (unused page)
        - MedicineInventory.jsx (unused page)

           date formatting helper in src/lib/.
