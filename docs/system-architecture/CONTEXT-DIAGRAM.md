# Context Diagram (DFD Level 0) — TechClinic System

The context diagram establishes a high-level overview of how the TechClinic: Health Record Management System interacts with its external entities. It represents the entire system as a single process — Process 0 — and identifies all actors and services that exchange data with the system, thereby defining the system boundary and scope.

Six external entities interact with the TechClinic System. The Student initiates the process by submitting registration data such as personal information, student ID, and department details through a public-facing kiosk interface; in return, the system provides registration confirmation and auto-fill data for returning students. The Nurse, as a clinic staff member, submits patient data and medicine dispensation records into the system, and receives patient records, low-stock alerts, outbreak notifications, analytics reports, and inventory information. The Doctor provides diagnosis data and login credentials, and receives access to patient records, alerts, and analytics data. The Admin, responsible for system oversight and user management, submits user activation and deactivation requests, personnel data, and login credentials, and receives a dashboard overview, activity logs, analytics summaries, alerts, and the personnel list. The Patient is the recipient of the system's outputs, namely medical certificates and prescriptions. Finally, Supabase, the external backend-as-a-service platform, provides authentication tokens and query results to the system while receiving authentication requests, data queries, and CRUD operations.

Together, the context diagram demonstrates that the TechClinic System communicates with four types of users (Student, Nurse, Doctor, and Admin), one output recipient (Patient), and one external service (Supabase). All data entering and leaving the system is accounted for, ensuring that the system boundary is clearly defined. This high-level representation serves as the foundation for the more detailed DFD Level 1, which decomposes Process 0 into its constituent sub-processes to reveal the internal data processing logic.
