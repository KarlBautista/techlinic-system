# System Flowchart — TechClinic System

The system flowchart illustrates the complete process flow of the TechClinic system. The process begins with the login verification, which ensures that only authorized personnel can access sensitive medical information. The credential verifier checks the entered username and password, displaying an error message for invalid login attempts. Once verified, the system checks if the account is active and redirects the user to the appropriate dashboard based on their role. The doctor's interface includes the dashboard, analytics, patient record, medicine inventory, notification, and settings. The nurse's interface includes the dashboard, analytics, new patient, patient record, medicine inventory, notification, and settings. The admin's interface provides user account management, activity logs, personnel list, and personnel oversight.

On the dashboard, users can view a list of recent patients with their diagnosis status (completed or pending). Incomplete records can be opened directly from the dashboard for diagnosis. The analytics section generates interactive graphical reports including most-used stock, patient record count, top diagnoses, and records per department, which can be filtered by time period and downloaded as image files. In the new patient section, the nurse collects and encodes patient information such as student ID, full name, email, and contact number. An auto-fill feature detects returning patients and populates their existing data. The nurse may complete a preliminary diagnosis or forward the record to the doctor for final assessment. The doctor then adds the diagnosis by selecting a disease from the catalog, entering treatment and medication, and marking the record as complete. If needed, the doctor generates a prescription or medical certificate that is sent directly to the student's email, since students do not have login credentials.

The patient record section serves as a centralized repository where users can search, view, and print patient records and medical histories. The medicine inventory section displays all available medications with a color-coded status system: green for sufficient stock, yellow for low stock, and red for critically low or near-expiry items. Users can add new medicines, update stock levels, or remove expired entries, with the system triggering low-stock notifications automatically. The notification section displays alerts for top diagnoses, rising patient counts, and medicine stock warnings. The settings section allows users to update personal information and change account credentials. On the admin side, the administrator can manage user accounts by activating or deactivating staff, view activity logs, and oversee the personnel list. All significant actions across the system are logged in the audit trail for accountability.
