# System Flowchart — TechClinic System

The system flowchart illustrates the four main process flows of the TechClinic system. The Student Registration Flow begins when a student opens the registration portal, enters their patient information (with auto-fill for returning students), and submits the form. The system stores the patient and visit records in the database and sends a registration notification to clinic staff, after which the student receives a confirmation.

The Staff Authentication Flow handles login for nurses, doctors, and admins. After entering credentials, the system validates them against Supabase Auth and checks whether the account is active. If authentication succeeds, the user is directed to their role-specific dashboard. The Clinical Workflow covers the core health record process: staff view the dashboard and patient records, select an incomplete record, and either the nurse creates an initial assessment (possibly deferring to the doctor) or the doctor directly adds a diagnosis with disease, treatment, and medication. Once the record is marked complete, the doctor may generate a prescription or medical certificate that is sent to the student's email. Every significant action is logged in the audit trail.

The Admin Workflow shows how administrators manage user accounts (activating or deactivating staff), view activity logs, and oversee the personnel list. The Inventory Management Flow covers medicine stock operations where nurses can add, update, or delete medicines, with the system automatically triggering a low-stock notification when levels fall below the threshold.
