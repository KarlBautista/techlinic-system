 Entity-Relationship Diagram (ERD) — TechClinic System

The entity-relationship diagram presents the data model of the TechClinic system, consisting of nine tables stored in Supabase PostgreSQL. The core clinical tables — patients, records, and diagnoses — capture the health record workflow where a student visits the clinic and a nurse or doctor creates a visit record, then adds one or more diagnoses referencing the diseases catalog. Students do not have login credentials; instead, completed prescriptions and medical certificates are sent directly to their email through the shared_documents table, which stores generated documents as JSONB with automatic 90-day expiration.

Only clinic staff interact with the system. The users table stores doctor and nurse profiles (including account status via `is_active`, `deactivated_at`, and a `signature_url` for digital signatures), while the audit_trail table logs every significant action linked to the responsible staff member through the `actor_id` foreign key. Both doctors and nurses can create records and diagnoses, though nurses may defer clinical assessments to doctors. The medicines table tracks pharmaceutical inventory independently, and the notifications table stores shared system-wide alerts such as low-stock warnings and outbreak notifications visible to all staff.

Four one-to-many relationships connect the entities: patients to records (`student_id`), records to diagnoses (`record_id`), diseases to diagnoses (`disease_id`), and users to audit trail entries (`actor_id`). All tables are protected by Row Level Security (RLS) policies at the database level, ensuring data access is restricted according to the role-based security model.
